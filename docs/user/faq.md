# EchoVault FAQ

> 更新日期：2026-07-09

---

## 通用问题

### Q: EchoVault 是免费的吗？

是的，EchoVault 是开源软件，基于 Apache 2.0 许可证发布。你可以免费使用、修改和分发。

### Q: EchoVault 需要联网吗？

不需要。EchoVault 采用离线优先设计，所有核心功能在无网络环境下也可使用。联网后可自动同步数据到其他设备。

### Q: EchoVault 支持哪些平台？

- **服务端：** 任何支持 Docker 的 Linux 服务器
- **客户端：** Android、iOS、Windows、macOS、Linux（Flutter 跨平台）

### Q: EchoVault 和网易云音乐 / QQ 音乐有什么区别？

EchoVault 不是流媒体服务，不提供在线音乐库。它是**你自己的音乐管理平台**：

- 你管理自己的音乐文件
- 你的数据存放在你自己的服务器上
- 无广告、无版权限制、无数据收集

## 安装部署

### Q: 部署 EchoVault 需要什么配置？

最低配置：1核 CPU、512MB 内存、1GB 磁盘。推荐配置：2核 CPU、1GB 内存、根据曲库大小准备磁盘。

### Q: 一定要用 Docker 吗？

Docker Compose 是最推荐的部署方式。你也可以手动编译运行 Go 后端，但不提供官方支持。

### Q: 支持 Windows 服务器吗？

推荐使用 Linux 服务器。Windows 上可以通过 WSL2 + Docker Desktop 运行。

### Q: 如何配置外网访问？

建议使用 Nginx/Caddy 反向代理 + Let's Encrypt 免费证书提供 HTTPS 访问。详见 [部署文档](../deployment/docker-compose.md#72-安全建议)。

## 曲库管理

### Q: 支持哪些音频格式？

支持 MP3、FLAC、M4A/AAC、OGG、WAV、OPUS。其中 MP3 和 FLAC 的元数据解析支持最完整。

### Q: 扫描后歌曲信息不正确怎么办？

扫描结果页面支持手动编辑元数据，可以修改标题、艺术家、专辑等信息后再发布。

### Q: 为什么有些歌曲没有封面？

封面提取取决于音频文件是否嵌入了封面图片。部分老旧的 MP3 文件可能没有嵌入封面。你可以在发布前手动添加封面。

### Q: 我的手机存储不够，可以只保留云端的歌曲吗？

可以。已发布的歌曲在服务器上有完整副本。你可以从本地删除文件，需要时再下载。

## 同步

### Q: 同步是实时的吗？

采用轮询机制，客户端定期检查服务器变更。在正常的网络环境下，延迟通常在 1-2 分钟内。

### Q: 如果我离线修改了同一首歌怎么办？

EchoVault 使用基于版本的"最后写入胜出"（last-write-wins）冲突解决策略。后修改的版本会覆盖先前的版本。

### Q: 同步会消耗大量流量吗？

增量同步仅传输变更的元数据（JSON），数据量很小。只有下载/上传音频文件时才会消耗较多流量。

## 常见错误

### Q: 连接服务器显示"连接超时"

**解决方法：**
1. 检查服务器是否在运行：`docker compose ps`
2. 检查防火墙是否开放了 8080 端口
3. 确认客户端填写的服务器地址和端口正确
4. 尝试用浏览器访问 `http://服务器IP:8080` 确认网络可达

### Q: 登录提示"用户名或密码错误"

**解决方法：**
1. 确认用户名和密码拼写正确（区分大小写）
2. 如果忘记密码，需要在服务器端重置（目前不支持自助找回）

### Q: 上传文件失败

**解决方法：**
1. 检查文件大小是否过大（目前无硬性限制，但超大文件可能超时）
2. 检查磁盘空间是否充足
3. 查看服务器日志：`docker compose logs echovault-server`
4. 检查网络连接是否稳定

### Q: 客户端闪退

**解决方法：**
1. 确保使用最新版本的客户端
2. 尝试清除应用缓存
3. 检查设备系统版本是否符合要求
4. 如果持续闪退，请提交 [GitHub Issue](https://github.com/inkOrCloud/EchoVault/issues)

## 安全

### Q: 我的音乐会泄漏吗？

EchoVault 的设计原则是数据由你完全掌控：
- 所有数据存储在你的私有服务器上
- 通信通过 JWT 认证，未登录的设备无法访问
- 建议配合 HTTPS 使用

### Q: JWT 密钥是什么？为什么重要？

JWT 密钥用于签发登录令牌。如果密钥泄露，攻击者可以伪造登录令牌访问你的服务器。生产环境必须使用高强度随机密钥。

### Q: 如何修改密码？

目前不支持在客户端修改密码。需要在服务器端直接操作数据库。未来版本将添加此功能。
