## 1. 简介

### 1.1 什么是 EchoVault？

EchoVault（音匣）是一个**混合模式音乐管理平台**，支持：

- 🎵 **本地音乐管理** — 扫描本机音频文件，建立离线曲库
- ☁️ **云端曲库同步** — 多设备间同步歌曲元数据、歌词、歌单
- 📤 **文件发布共享** — 手动上传音频文件到服务器，供其他设备使用

### 1.2 核心概念

| 概念 | 说明 |
|:---|:---|
| **曲库** | 所有歌曲的集合，包括本地扫描和服务器下载的歌曲 |
| **歌单** | 用户自定义的歌曲分组，可跨设备同步 |
| **同步** | 多设备间自动同步曲库、歌单和歌词数据 |
| **设备** | 安装 EchoVault 客户端的每台设备 |

## 2. 快速开始

### 2.1 首次打开

1. 启动 EchoVault 客户端，首先显示**服务器配置**页面
2. 输入你的 EchoVault 服务器地址（例如 `http://192.168.1.100:8080`）
3. 点击"连接测试"，成功后会跳转到登录页面

### 2.2 注册账号

1. 在登录页面点击"注册"
2. 填写用户名、显示名称、密码
3. 点击注册，成功后自动登录

### 2.3 添加音乐

1. 进入"曲库"页面
2. 点击右下角的"+"按钮
3. 选择"扫描本地文件"
4. 选择音频文件或目录
5. 扫描完成后，歌曲将显示在曲库中

## 3. 曲库管理

### 3.1 查看曲库

曲库页面以列表形式展示所有歌曲，每首歌曲显示：

- 歌曲标题
- 艺术家
- 专辑
- 时长
- 文件来源（本地/已上传/云端）

### 3.2 扫描本地音乐

支持两种扫描方式：

**方式一：快速扫描文件**

选择单个或多个音频文件进行扫描，适合少量歌曲。

**方式二：扫描目录**

选择整个音乐目录，系统会自动递归扫描所有支持的音频格式。

支持的格式：MP3、FLAC、M4A、OGG、WAV、OPUS

### 3.3 歌曲元数据

扫描时自动提取以下元数据：

| 字段 | 说明 |
|:---|:---|
| 标题 | 从文件标签读取 |
| 艺术家 | 从文件标签读取 |
| 专辑 | 从文件标签读取 |
| 流派 | 从文件标签读取 |
| 音轨号 | 从文件标签读取 |
| 年份 | 从文件标签读取 |
| 时长 | 从文件编码信息读取 |
| 采样率 | 从文件编码信息读取 |
| 比特率 | 从文件编码信息读取 |

### 3.4 发布到服务器

扫描后的歌曲可以"发布"到服务器：

1. 在扫描结果页面查看解析后的元数据
2. 可以手动编辑修正元数据
3. 点击"发布"将歌曲文件上传到服务器
4. 发布成功后，其他设备可以通过同步获取该歌曲

## 4. 音乐播放

### 4.1 播放控制

点击任意歌曲即可开始播放。播放器提供：

- ▶️ / ⏸ 播放/暂停
- ⏭ 下一首 / ⏮ 上一首
- 🔊 音量控制
- 📋 播放列表查看

### 4.2 播放模式

| 模式 | 说明 |
|:---|:---|
| 顺序播放 | 按当前列表顺序播放 |
| 单曲循环 | 重复当前歌曲 |
| 随机播放 | 随机选取歌曲 |

### 4.3 迷你播放器

在曲库或歌单页面底部，显示迷你播放器：

- 当前歌曲信息（标题 + 艺术家）
- 播放/暂停按钮
- 点击可展开为全屏播放器

### 4.4 全屏播放器

全屏播放器提供：

- 专辑封面展示
- 同步歌词显示
- 进度条拖动
- 完整播放控制

## 5. 歌单管理

### 5.1 创建歌单

1. 进入"歌单"页面
2. 点击"新建歌单"
3. 输入歌单名称
4. 创建成功

### 5.2 管理歌单歌曲

**添加歌曲到歌单：**

1. 打开歌单详情页
2. 点击"添加歌曲"
3. 搜索或浏览曲库
4. 选择要添加的歌曲

**移除歌曲：**

在歌单详情页，长按或点击歌曲旁的移除按钮。

**调整顺序：**

拖拽歌曲右侧的排序手柄调整播放顺序。

### 5.3 删除歌单

在歌单列表页，长按歌单或点击菜单选择"删除"。

## 6. 歌词功能

### 6.1 查看歌词

播放歌曲时，如果服务器存在对应歌词，会在全屏播放器中自动显示。

### 6.2 歌词格式

支持标准 LRC 格式歌词：

```
[00:12.00] 陪你熬夜聊天到爆肝也没关系
[00:16.00] 陪你逛街逛到扁平足也没关系
[00:20.00] 超感谢你让我重生 整个新世界
```

同时支持翻译歌词（原歌词与翻译对照显示）。

### 6.3 搜索歌词

系统会自动按歌曲标题和艺术家搜索匹配的歌词。用户也可以手动搜索。

## 7. 文件发布与共享

### 7.1 上传流程

1. 选择音频文件（或扫描目录）
2. 系统解析元数据并显示预览
3. 可编辑修正元数据
4. 上传文件到服务器
5. 服务器计算文件哈希（SHA256），避免重复上传

### 7.2 文件去重

系统通过 SHA256 文件哈希自动去重：

- 同一设备多次扫描同一文件 → 只保留一条记录
- 多设备扫描相同文件 → 自动关联到同一条记录

### 7.3 文件状态

| 状态 | 说明 |
|:---|:---|
| `local_only` | 仅存在于本地设备，未上传 |
| `uploaded` | 已上传到服务器 |
| `downloaded` | 从服务器下载到本地 |
| `cloud_only` | 仅存在于服务器（其他设备上传的） |

## 8. 设备管理

### 8.1 查看设备

在"设备"页面查看所有已注册设备：

- 设备名称（可重命名）
- 设备平台（Android/iOS/Windows/macOS/Linux）
- 最后同步时间

### 8.2 设备注册

首次登录时，当前设备会自动注册。每台设备用唯一的设备 ID 标识。

### 8.3 移除设备

在设备列表页，点击"移除"将设备与账号解绑。解绑后该设备需要重新登录。

## 9. 同步功能

### 9.1 同步概览

EchoVault 使用"离线优先"同步策略：

```
离线操作 ──► 本地存储 ──► 联网后 ──► 增量同步到服务器 ──► 推送到其他设备
```

- ✅ 离线时所有功能可用
- ✅ 联网后自动同步，无需手动操作
- ✅ 增量同步，仅传输变更数据

### 9.2 同步内容

| 数据类型 | 同步范围 |
|:---|:---|
| 歌曲元数据 | 所有设备间同步 |
| 歌单 | 所有设备间同步 |
| 歌词 | 所有设备间同步 |
| 音频文件 | 需手动上传/下载（文件较大） |

### 9.3 同步状态指示器

主页面顶部的同步图标表示当前同步状态：

| 图标 | 说明 |
|:---|:---|
| ✅ | 同步完成，数据最新 |
| 🔄 | 正在同步中... |
| ⚠️ | 同步异常，需检查网络 |
| ❌ | 同步失败 |

### 9.4 手动同步

在同步状态页面，可以：

- 查看最近的同步历史
- 手动触发"立即同步"
- 查看同步错误详情
